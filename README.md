# VinylA Collection 💿

**실물 LP를 위한 프라이빗 아카이빙 서비스**

VinylA(바이닐라)는 "바닐라(Vanilla, 순정)"와 "바이닐(Vinyl)"을 합친 이름입니다. 스트리밍 플레이리스트가 아니라, 실제로 손에 쥔 LP 재킷과 알맹이, 그 안의 수록곡을 있는 그대로 기록하고 자산으로 남기기 위한 서비스입니다. 카메라로 앨범 커버를 한 번 비추면 자동으로 정보가 채워지고, 웹과 모바일 어디서든 같은 컬렉션을 확인할 수 있습니다.

- **웹**: https://vinyla-collection.com
- **모바일 앱**: iOS / Android (Expo 기반)

이 문서는 실제로 서비스를 이용하는 사용자와 클라이언트를 위해, 화면별로 무엇을 할 수 있는지 안내합니다. 개발/배포 관련 정보는 맨 아래 [개발자를 위한 안내](#-개발자를-위한-안내) 섹션에 따로 정리했습니다.

---

## 🧭 이용 흐름

1. 구글(또는 애플) 계정으로 로그인
2. 처음 한 번, 닉네임과 관심 장르를 간단히 설정
3. 카메라로 LP 커버를 스캔하거나, 검색으로 앨범을 찾아 보관함/위시리스트에 추가
4. 마이페이지에서 컬렉션 통계를 확인하고, 링크나 이미지로 다른 사람과 공유

## 🌐 웹 서비스 화면 안내

### 랜딩 페이지
처음 접속하면 나오는 소개 화면입니다. 서비스 소개와 함께 구글 계정으로 로그인할 수 있는 버튼이 있습니다.

### 최초 설정
로그인 후 처음 한 번만 나타나는 화면으로, 닉네임과 관심 장르를 선택합니다. 이미 설정을 마친 사용자는 이 화면을 다시 거치지 않습니다.

### 보관함 (컬렉션)
로그인 후 기본으로 보게 되는 나의 LP 진열장입니다.
- 커버 아트워크 중심의 갤러리형 그리드(4열/조밀 6열) 또는 표 형태(테이블) 중 원하는 보기 방식을 고를 수 있습니다.
- 최신순 · 오래된순 · 가나다순 · 출시연도순으로 정렬할 수 있고, 보유한 장르 태그로 필터링할 수 있습니다.
- 앨범을 클릭하면 상세 팝업이 열려 트랙리스트, 시장 추정가, 실제 구입가, YouTube 전곡 듣기, Discogs 검색 링크를 확인할 수 있습니다.
- 우측 상단 공유 아이콘으로 보관함 전체를 이미지 또는 링크로 공유할 수 있습니다.

### 위시리스트
아직 갖고 있지 않지만 갖고 싶은 앨범을 따로 모아두는 공간입니다. 보관함과 동일한 보기/정렬 방식을 제공하며, 언제든 원하는 앨범을 실제 보관함으로 옮길 수 있습니다.

### 검색
Discogs의 방대한 LP 데이터베이스를 실시간으로 검색합니다. 검색어가 없을 때는 장르별 탐색 그리드가 노출되어, 관심 있는 장르를 클릭하는 것만으로도 관련 앨범을 둘러볼 수 있습니다.

### 마이페이지
나의 프로필과 컬렉션 통계를 확인하고 관리하는 공간입니다.
- 시장 추정가 합계, 실제 지출액(공개 여부 선택 가능), 보유 LP 수, 관심 장르 등 컬렉션 분석 카드를 제공합니다.
- 컬렉션 수치(보유 장수, 최고가, 장르 편중도 등)에 따라 자동으로 **컬렉터 호칭(배지)** 을 획득하고 프로필에 장착할 수 있습니다. 일부 희귀 호칭은 조건이 공개되지 않은 채 `???`로 표시되어 있습니다.
- 대표 LP를 하나 지정해 프로필 상단에 크게 보여줄 수 있습니다.
- 최근 수집한 앨범을 타임라인 형태로 확인할 수 있습니다.
- 프로필/컬렉션 링크를 다른 사람에게 공유할 수 있고, 닉네임·프로필 사진·회원 탈퇴 등 계정 설정도 이곳에서 관리합니다.

### 공유 프로필 (퍼블릭 대시보드)
마이페이지에서 만든 공유 링크를 열면 누구나(로그인 없이도) 볼 수 있는 읽기 전용 프로필입니다. 보유 LP 수, 관심 장르, 대표 LP, 최근 수집 기록, 보관함/위시리스트 목록을 볼 수 있으며, 앨범 상세를 보려면 로그인이 필요합니다.

### 문의하기
서비스 이용 중 불편한 점이나 제안 사항을 남길 수 있는 공간입니다. 불만/건의/버그 신고/기타로 분류해 남길 수 있고, 답변이 달리면 이어서 확인·재문의할 수 있습니다.

## 📱 모바일 앱 화면 안내

### 온보딩
앱을 처음 실행하면 3단계로 서비스를 소개합니다 — 순정 아카이빙 철학, 카메라 스캔 기능, 그리고 로그인. 구글 또는 애플 계정으로 로그인할 수 있습니다.

### 홈 (보관함)
웹의 보관함과 동일한 나의 LP 진열장입니다. 그리드/테이블 보기 전환과 정렬을 지원하며, 하단 공유 버튼으로 보관함 이미지를 인스타그램 스토리로 바로 공유할 수 있습니다.

### 위시
갖고 싶은 앨범을 모아두는 공간으로, 웹의 위시리스트와 동일하게 동작합니다.

### 스캔
이 앱의 핵심 기능입니다. 카메라로 LP 커버를 비추고 촬영하면, AI가 이미지를 분석해 아티스트/앨범명을 추론한 뒤 Discogs에서 일치하는 앨범 후보를 찾아 보여줍니다. 후보 중 원하는 앨범을 선택해 바로 보관함이나 위시리스트에 추가할 수 있습니다. 갤러리에서 사진을 골라 스캔하는 것도 가능합니다.

### 검색
웹과 동일하게 장르 탐색과 Discogs 검색을 제공하며, 스크롤을 내리면 검색 결과가 계속 이어서 로드됩니다.

### 마이
웹 마이페이지와 동일한 기능(통계, 호칭, 대표 LP, 프로필 설정, 공유, 로그아웃/회원 탈퇴)을 모바일에 맞게 제공합니다.

## ✨ 핵심 기능 하이라이트

- **카메라 스캔 등록**: 앨범 커버 사진 한 장으로 아티스트/앨범명/트랙리스트/장르까지 자동 인식
- **Discogs 실시간 연동**: 전 세계 LP 데이터베이스에서 정확한 앨범 정보와 시장 추정가 확인
- **YouTube 청음 연동**: 상세 팝업에서 바로 전곡 듣기로 이동
- **이미지/링크 공유**: 보관함, 위시리스트, 개별 앨범을 인스타그램 스토리용 이미지나 링크로 공유
- **컬렉터 호칭(배지) 시스템**: 컬렉션 규모와 취향에 따라 자동으로 해금되는 게이미피케이션 요소
- **웹 ↔ 모바일 실시간 동기화**: 한쪽에서 앨범을 추가하면 다른 기기에서도 새로고침 없이 즉시 반영

## 👨‍💻 개발자를 위한 안내

### 기술 스택

- **Monorepo**: Turborepo 기반
- **Web App**: Next.js 16 (App Router, Turbopack)
- **Mobile App**: React Native (Expo)
- **Shared API/Logic**: `@vinyla/core-api` 패키지
- **Database / Auth**: Supabase (PostgreSQL + RLS)
- **External APIs**: Discogs API, Google Gemini(스캔 이미지 분석), YouTube Data API

### 시스템 아키텍처 및 최적화 내역

#### 1. 보안 (Supabase RLS)
VinylA는 사용자 간 데이터 격리를 보장하기 위해 강력한 Row Level Security (RLS) 정책을 채택하고 있습니다.
- `USER_VINYL` · `PROFILES`: 자신의 데이터만 추가/조회/수정/삭제 가능
- `ALBUM_MASTER` · `VINYL_TAG`: 공개 읽기 + 로그인 사용자 쓰기 (관리자만 삭제)
- 2026-07-11 보안 하드닝 마이그레이션으로 닉네임 30일 쿨다운 DB 트리거화, EVENT_LOG 검증/anon 플러딩 방지, USER_VINYL 무결성 제약(중복 방지·FK·CHECK), INQUIRY 상태 자동 재오픈 트리거, avatars 스토리지 정책 코드화 완료. 자세한 내용은 `supabase_schema.sql` 하단 및 `VinylA_Blue_Print.md` §4 참고.

#### 2. 백엔드 성능 최적화 (RPC)
관리자 대시보드 등의 무거운 통계/집계 쿼리에서 발생하는 클라이언트-서버 간 불필요한 트래픽 및 JS 메모리 점유율 폭발을 방지하기 위해 Postgres 내장 RPC(Remote Procedure Call) 함수 `get_user_vinyl_counts` 등을 통해 데이터를 효율적으로 조회합니다.

#### 3. 클라이언트 성능 최적화
모든 이미지 리소스(`next/image` 및 Expo Image)는 레이지 로딩과 캐싱을 통해 렌더링 성능을 극대화하며, 불필요한 마이그레이션 백그라운드 작업을 제거하여 앱 구동 속도를 높였습니다.

#### 4. API 인프라
스캔(VLM 매칭) 및 Discogs/YouTube 외부 조회 요청은 별도 Node.js 서버가 아니라 Next.js API 라우트(`apps/web/src/app/api/scan`, `apps/web/src/app/api/external/*`)로 처리됩니다. 모바일 앱은 프로덕션에서 배포된 웹 앱 도메인을 API 베이스 URL로 사용합니다.

### 🛑 에러 코드 (Error Codes) 체계

사용자 경험 향상 및 고객 문의 대응을 위해 명확한 고유 에러 코드 체계를 사용합니다. 앱 또는 웹에서 에러 팝업/토스트에 노출되는 에러 코드의 의미는 아래와 같습니다.

#### 🔐 AUTH (인증 에러)
- `AUTH-001`: 로그인이 필요한 기능에 접근했습니다 (세션 만료 등).
- `AUTH-002`: 사용자 정보를 불러올 수 없습니다.

#### 🗄️ DB (데이터베이스 에러)
- `DB-001`: 데이터 저장 중 오류가 발생했습니다. (고유 제약조건 위반 또는 권한 부족)
- `DB-002`: 데이터를 불러오는 중 오류가 발생했습니다.
- `DB-003`: 데이터 삭제 중 오류가 발생했습니다.
- `DB-004`: 상세 정보 업데이트 중 오류가 발생했습니다.

#### 🌐 EXT (외부 API 에러)
- `EXT-001`: Discogs 서버 연결이 지연되고 있습니다. (Rate Limit 또는 서비스 장애)
- `EXT-002`: Apple Music 서버 연결이 지연되고 있습니다.
- `EXT-003`: YouTube 검색 한도를 초과했습니다.
- `EXT-004`: 외부 서버 요청 시간이 초과되었습니다.

#### 📶 NET (네트워크 에러)
- `NET-001`: 네트워크 연결 상태가 불안정하거나 오프라인 상태입니다. (저장 시 로컬 임시 보관함에 Fallback 처리됩니다.)

#### ⚙️ SYS (시스템 에러)
- `SYS-001`: 알 수 없는 시스템 오류가 발생했습니다.

### 📦 실행 방법

```bash
# 의존성 설치
npm install

# 전체 패키지 빌드
npm run build

# Web 로컬 개발 서버 실행
npm run dev --filter=web

# Mobile (Expo) 로컬 개발 서버 실행
npm run dev --filter=mobile
```
